| CFR / 反事实遗憾最小化 | Counterfactual Regret Minimization，一种博弈论中的迭代算法，通过最小化反事实遗憾值来逼近纳什均衡 |

| Worker 进程 / 子进程 | 在 CPU 上并行执行游戏树遍历、样本收集及可选特征预处理的子进程 |

| 子进程队列 | 子进程与主进程之间的多进程安全队列；子进程将样本放入队列，主进程从队列取出样本并写入经验缓冲区 |
| 优势样本 | 包含信息状态、遗憾值（及可选动作）等用于训练优势网络的数据单元 |

| 信息状态 | 博弈中某玩家在某一时刻可观测到的状态表示，用于输入神经网络；可包含原始编码与人工设计的先验特征 |
| 先验特征 / 先验知识特征 | 基于博弈规则与领域知识人工设计并拼接到信息状态上的特征（如牌力、游戏阶段、下注统计等） |

## 1、本发明要解决的技术问题是什么？

本发明要解决的技术问题可从四个方面概括，便于代理人带着问题阅读下文：

1. **并行训练**：在深度反事实遗憾最小化（DeepCFR）算法中，游戏树遍历（CPU 密集）与神经网络训练（GPU 密集）若在同一进程内串行执行，会导致 CPU 与 GPU 无法同时满载，训练效率低。如何设计一种并行训练架构，使遍历与训练解耦、并行执行，从而显著提高训练吞吐量？

2. **经验缓冲区的样本来源**：传统实现中，经验缓冲区中的样本由单进程本地遍历游戏树后直接写入。当采用多子进程并行遍历时，若仍由主进程本地写缓冲区，则无法利用子进程产生的样本；若由子进程直接写共享缓冲区，则存在锁竞争与同步复杂度。如何设计一种“子进程通过队列向主进程供给样本、主进程再将样本写入经验缓冲区”的方法，使缓冲区的样本来源由队列供给替代本地产生，且队列或缓冲区满时采用**一部分 FIFO、一部分随机替换**的混合替换策略，兼顾时序性与多样性、避免阻塞？

3. **输入特征与训练目标尺度**：原始信息状态中的金额类特征及优势网络的学习目标（遗憾值）数值范围随游戏配置（筹码、盲注）和牌局进程变化大，若不做归一化或仅用固定参数，会导致训练不稳定。如何将博弈领域的先验知识（牌力、阶段、下注统计等）设计为可拼接的特征，并从游戏配置自动解析归一化基准，对输入与目标做尺度归一化与裁剪，以提升训练稳定性？

4. **策略网络（及优势网络）训练的采样分布**：从经验缓冲区随机采样时，若新近产生的样本或高重要性样本占比过低，训练数据分布会严重偏向历史策略或低价值样本，导致分布漂移、训练抖动。如何设计一种**多维度分层采样**方法（如按时间/迭代次数划分新近与历史样本，按重要性如遗憾值幅度、对策略更新贡献度等划分高/低重要性层），在每次训练时按比例从各层抽取或采用重要性加权的分层采样，保证新近样本与高重要性样本占比、减轻分布漂移并提升样本利用效率？

## 2、详细介绍技术背景，并描述已有的与本发明最相近似的实现方案

### 2.1 作为本发明基础的公知技术

- **DeepCFR 算法**：深度反事实遗憾最小化通过神经网络近似策略与优势函数。每轮迭代中，对博弈树进行遍历（如 Monte Carlo 采样），利用当前优势网络进行动作采样并计算反事实值与遗憾值，生成优势样本与策略样本，再使用这些样本更新策略网络与优势网络。因此训练过程同时包含“游戏树遍历（主要在 CPU）”和“网络训练（主要在 GPU）”两类任务。

- **经验缓冲区与采样**：在 DeepCFR 及类似算法中，通常将遍历得到的样本存入经验缓冲区（Replay Buffer、Reservoir 等），训练时从缓冲区采样得到批次再更新网络。传统单进程实现中，样本由本进程遍历后直接写入缓冲区，即“本地产生、本地写入”。

- **多进程与队列**：单机多核环境下，可通过多进程（如 Python 的 multiprocessing）分配任务；进程间通过队列（Queue）传递数据，实现生产者-消费者式的并行。

- **应用场景**：本发明特别适用于**德州扑克（Texas Hold'em）**及同类不完全信息博弈（如限注/无限注、单挑/多人）。以下 2.2 与 2.3 节中单独列出与**德州扑克/扑克 AI** 相关的专利与文献，便于代理人做针对性对比。

### 2.2 与本发明最相近似的现有实现方案

**方案 A：通用分布式强化学习训练**  
- **专利/文献**：专利号 US10445641B2（DeepMind）。  
- **结构**：Actor-Learner 架构；多个 Actor 与环境交互生成经验元组，Learner 从缓冲区采样训练，参数通过 Parameter Server 同步。  
- **方法**：Actor 执行“环境交互→采集数据→写入缓冲区”；Learner 从缓冲区采样→更新网络→参数经 Parameter Server 同步至 Actor。  
- **与本发明的关系**：同属“数据采集与模型训练解耦”，但面向通用强化学习（环境交互），非博弈树遍历；且采用 Parameter Server，未涉及“子进程队列向经验缓冲区供给样本”及“分层采样”“先验特征设计”等。

**方案 B：基于 Ray 的 DeepCFR 开源实现**  
- **专利/文献**：开源实现，无专利号；可注明参考仓库如 David-Flurin/DeepCFR、EricSteinberger/Deep-CFR 等。  
- **结构**：使用 Ray 框架，存在 Learner、Actor 等角色；数据与任务通过 Ray 的远程调用与对象存储传递。  
- **方法**：Actor 负责采样（含博弈环境推进与样本产生）；Learner 负责训练；样本与参数的传递依赖 Ray 的分布式机制。  
- **与本发明的关系**：同属 DeepCFR 并行化，但并行框架为 Ray，非单机多进程 + 队列；且未明确描述“子进程队列替代经验缓冲区样本来源”“队列满时一部分 FIFO、一部分随机替换的混合策略”“多维度分层采样（时间+重要性）”“输入特征先验设计与从配置解析归一化基准”等。

**方案 C：GPU 加速 CFR（如 GPUCFR）**  
- **专利/文献**：学术/技术文献或实现，非专利；若检索到对应专利号或论文出处可在此补充。

- **方法**：算法层面的并行化（信息集上的矩阵运算），不涉及“多进程游戏树遍历 + 独立 GPU 训练进程”的架构。

**方案 P：基于虚拟遗憾最小化算法的德州扑克 AI 训练方法（CN112691383A）**  
- **专利/文献**：中国专利申请 CN112691383A（上海交通大学，2021 年 1 月申请，2021 年 4 月公开；发明人：张轶飞、程帆、张冬梅）。  
- **结构**：一种基于虚拟遗憾最小化算法（即 CFR/反事实遗憾最小化）的德州扑克 AI 训练方法；包括游戏特征抽象（私人手牌与公共牌、9-Buckering 牌力映射）、针对特定玩家的策略预测神经网络模型、虚拟遗憾最小化算法以该策略预测模型为对手训练得到 AI 行为策略、以及与真实玩家对局并记录。策略预测神经网络为卷积+池化+全连接（如 1326/169 节点），输入为牌力值与游戏界面信息，输出为抽象特征。  
- **方法**：步骤 1）获取手牌与公共牌信息并做游戏特征抽象（含 9-Buckering 牌力值）；步骤 2）基于玩家历史对局建立针对该玩家的策略预测神经网络模型（Q-learning 或策略梯度框架），并对玩家每步行为预测、对局结束后用实际行为微调；步骤 3）采用虚拟遗憾最小化算法，以玩家策略预测神经网络为对手，在博弈树上计算虚拟价值与虚拟遗憾值，用后悔匹配机制得到 AI 行为策略；步骤 4）用训练得到的 AI 与真实玩家对局并记录。说明书中将算法表述为结合蒙特卡洛 CFR 与纯 CFR 的改进。  
- **与本发明的关系**：同属**德州扑克 + 遗憾最小化（CFR）+ 神经网络**，应用场景高度相关，需重点对比。区别包括：（1）CN112691383A 为单进程训练流程，未公开多子进程并行、子进程队列向主进程供给样本再写入经验缓冲区、队列或缓冲区满时一部分 FIFO 一部分随机替换的混合策略；（2）未公开从经验缓冲区的多维度分层采样（时间+重要性，新近/高重要性样本占比）；（3）以“针对特定玩家的策略预测神经网络”为对手进行 CFR 训练，与本发明中多 Worker 游戏树遍历生成优势/策略样本、主进程从缓冲区采样训练策略/优势网络不同；（4）游戏抽象方式为 9-Buckering 牌力与 CNN 抽象特征，与本发明从游戏配置解析归一化基准、先验特征设计与多阶段归一化不同。代理人应在说明书中单独对比 CN112691383A 并明确上述区别。

**方案 O：德州扑克/扑克 AI 相关文献（无专利号，供对比）**  
- **DeepStack**（Science 2017，阿尔伯塔大学）：单挑无限注德州扑克，递归推理 + 分解 + 深度直觉网络，实时重解；未涉及 DeepCFR、子进程队列、经验缓冲区供给或分层采样。  
- **Libratus**（Science 2017，CMU）：单挑无限注德州扑克，抽象 + 蓝图策略 + 子博弈求解；基于 CFR 等，但为离线求解与实时求解架构，非“多子进程队列 + 经验缓冲区 + 分层采样”的 DeepCFR 训练流程。  
- **Pluribus**（Science 2019，Facebook/CMU）：六人无限注德州扑克，自对弈与实时改进；未公开与本发明相同的“子进程队列替代经验缓冲区 + 分层采样 + 先验特征”的 DeepCFR 训练架构。  
- **DeepHoldem / Poker-CNN 等**（IEEE、AAAI 等论文）：德州扑克或扑克变体的端到端/模式学习；未涉及 CFR 游戏树遍历、子进程队列供给缓冲区或按迭代次数的分层采样。  
- **与本发明的关系**：同属德州扑克/扑克 AI，但算法与架构不同；本发明针对 DeepCFR 的并行训练、队列供给经验缓冲区、先验特征与分层采样，可与上述文献在“应用场景相同、技术方案不同”层面做对比。

## 3、以因果关系推理的方式推导出现有技术的缺点是什么？针对这些缺点，说明本发明的目的。

**推理一（并行训练）**  
- 单进程 DeepCFR 中，游戏树遍历与网络训练串行执行，CPU 与 GPU 无法同时满载，训练吞吐量受限于单核遍历速度。  
- 若采用通用分布式架构（如 Actor-Learner + Parameter Server），其面向“环境交互”而非“博弈树遍历”，且 Parameter Server 增加复杂度和同步开销；现有 DeepCFR 的 Ray 实现依赖另一套框架，未明确给出“单机多进程 + 队列”下 CPU 遍历与 GPU 训练的解耦方式。  
- **发明目的**：提供一种面向 DeepCFR 的并行训练架构，主进程专注 GPU 训练与缓冲区管理，多个子进程专注 CPU 上的游戏树遍历，通过队列与共享缓冲区连接，使遍历与训练并行执行，提高训练效率。

**推理二（经验缓冲区样本来源）**  
- 传统上经验缓冲区的样本由单进程本地遍历后直接写入；多进程下若由子进程直接写共享缓冲区，锁竞争与同步复杂。  
- 若没有明确的“子进程→队列→主进程→缓冲区”的数据流，则无法在保持主进程逻辑简单的前提下，将子进程产生的样本稳定、可控地注入经验缓冲区；若队列满时仅采用单一 FIFO 或单一随机替换，则要么过度偏向时序、要么丢失近期样本的连续性。  
- **发明目的**：提供一种“子进程队列替代经验缓冲区样本来源”的方法：子进程将样本放入队列，主进程从队列取出样本再写入经验缓冲区；队列满时在子进程侧采用**一部分 FIFO、一部分随机替换**的混合替换策略，兼顾时序性与多样性、避免阻塞；经验缓冲区若已满也可采用混合淘汰（一部分 FIFO、一部分随机替换），使经验缓冲区的样本由队列供给而非本地产生。

**推理三（输入特征与训练目标尺度）**  
- 信息状态中的金额特征及优势网络的学习目标（遗憾值）数值范围随游戏配置与牌局进程变化大，固定或手动归一化难以兼顾多配置与极端情况，易导致训练不稳定。  
- 仅使用原始信息状态、不引入博弈领域先验（如牌力、阶段、下注统计），会加大网络学习难度与样本需求。  
- **发明目的**：提供一种输入特征与目标尺度的设计方法：从游戏配置自动解析归一化基准（如最大筹码）；设计并拼接基于博弈规则的先验特征（如牌力、游戏阶段、下注统计等）；对遗憾值等目标按基准归一化并可选裁剪；使输入与目标尺度稳定、先验知识显式参与训练，提升稳定性与收敛性。

**推理四（采样分布与分布漂移）**  
- 从经验缓冲区纯随机采样时，新近产生的样本或高重要性样本可能占比很低，训练数据分布严重偏向历史策略或低价值样本，导致分布漂移、训练抖动甚至发散。  
- **发明目的**：提供一种**多维度分层采样**方法：除按时间/迭代次数将样本分为新近样本与历史样本外，还可按重要性（如遗憾值幅度、对策略更新贡献度等）划分高/低重要性层；按设定比例从各层分别抽取再合并打散，或采用重要性加权的分层采样，用于策略网络与优势网络的训练批次，保证新近样本与高重要性样本占比，减轻分布漂移并提升样本利用效率。

## 4、本发明技术方案的详细阐述（结合流程图、原理框图说明）

以下按四条主线阐述，建议配套附图：**系统架构图**（主进程 / 子进程 / 队列 / 经验缓冲区）、**主进程训练与收集流程图**、**子进程遍历与入队流程图**、**多维度分层采样流程图**（含时间维度与重要性维度）、**先验特征与归一化框图**。附图中关键词与注释尽量使用中文，附图以黑白方式提供。

### 4.1 并行训练架构

- **主进程**：运行于单机，主要使用 GPU。负责从经验缓冲区采样得到训练批次；使用 GPU 对策略网络与各玩家优势网络进行训练与参数更新；将更新后的网络参数序列化后通过“参数同步队列”下发给各子进程；管理经验缓冲区（从队列接收样本并写入、从缓冲区采样）；可选地执行负载均衡、动态批次大小调整；定期检查各子进程是否存活；保存检查点等。主进程不执行游戏树遍历。

- **多个子进程（Worker）**：每个子进程仅使用 CPU。负责从参数同步队列接收最新网络参数并更新本地网络副本；使用本地副本进行游戏树遍历（Monte Carlo 采样），按 DeepCFR 算法生成优势样本与策略样本；可选地在本地对信息状态做特征预处理或归一化；将样本批量放入“样本队列”（优势样本可按玩家分队列）。可选地，子进程定期检查主进程是否存活，主进程退出时子进程自动退出。

- **队列与经验缓冲区**：  
  - 样本队列：每个玩家一个优势样本队列（或所有优势样本共用一个队列）、一个策略样本队列；子进程将样本批量放入，主进程从队列批量取出。  
  - 参数同步队列：主进程将序列化后的网络参数放入，各子进程从中取参数并更新本地网络。  
  - 经验缓冲区：由主进程管理（可为 Reservoir、RandomReplacement 等结构）；主进程从样本队列取出的样本写入经验缓冲区；训练时从经验缓冲区采样得到批次。  

数据流：子进程遍历→产生样本→放入样本队列；主进程从样本队列取样本→写入经验缓冲区→从经验缓冲区分层采样→GPU 训练→将新参数放入参数同步队列；子进程取参数后更新本地网络，继续遍历。从而实现“遍历与训练”的流水线并行。

### 4.2 子进程队列替代经验缓冲区样本的方法

- **样本来源替代**：经验缓冲区中的样本不再由主进程本地遍历产生后直接写入，而是由子进程遍历产生后放入样本队列，主进程从样本队列取出再写入经验缓冲区。即“经验缓冲区的样本来源”由“本地产生”变为“子进程通过队列供给”。

- **队列组织**：可为每个玩家维护一个优势样本队列，策略样本使用一个或多个策略队列。子进程每产生一批优势样本或策略样本，将该批放入对应队列；主进程在每轮迭代中，对每个优势队列和策略队列执行“批量 get（如 get_nowait 循环直至队列空或达到本次最大收集量）→ 将取出的样本逐个或批量 add 到对应经验缓冲区”，并在写入时可由主进程标记当前迭代次数（便于后续分层采样时区分新/老样本）。

- **队列满时的混合替换策略**：样本队列设有最大长度（maxsize）。当子进程向队列放入一批样本时，若队列已满，采用**一部分 FIFO、一部分随机替换**的混合策略：例如按设定比例或随机决策，一部分情况下丢弃队列中最旧的一批（FIFO），另一部分情况下随机选择队列中某一位置或某一批丢弃后再放入当前批（随机替换），从而兼顾时序性与多样性、避免子进程在 put 时阻塞、也避免主进程因队列无限增长而延迟。主进程侧无需再对队列做“清空”或“丢弃”操作，仅需按需从队列取样本并写入缓冲区。经验缓冲区在写入时若已满，也可采用混合淘汰策略：一部分容量或一定比例按 FIFO 淘汰最旧样本，另一部分按随机替换（随机选择槽位覆盖），以平衡新样本占比与历史分布多样性。

- **批量收集与迭代标记**：主进程从队列收集样本时，可设定本次最大收集量（如根据当前缓冲区大小与训练批次大小动态调整），批量 get 后批量 add 到经验缓冲区；在 add 时使用主进程的当前迭代次数标记每个样本的“迭代”字段，保证分层采样时能正确区分新样本与老样本。

### 4.3 输入特征先验知识与设计

- **从游戏配置解析归一化基准**：从游戏配置字符串（如包含 `stack=...`、`blind=...` 的字符串）中解析各玩家初始筹码，取最大值作为最大筹码 max_stack；若解析失败则使用默认值。max_stack 作为金额特征归一化、以及优势目标（遗憾值）尺度归一化的基准。

- **先验特征的设计与拼接**：在原始信息状态（如 OpenSpiel 提供的信息状态张量）基础上，增加基于博弈规则与领域知识的人工特征并拼接，形成扩展信息状态再输入策略网络与优势网络。先验特征可包括但不限于：  
  - 起手牌强度（如基于预计算的起手牌强度表）、当前手牌强度（考虑公共牌后的成牌强度）；  
  - 是否中牌、成牌类型（对子、两对、三条、顺子、同花、葫芦、四条、同花顺等）的布尔或数值编码；  
  - 听牌相关（如同花听牌、顺子听牌及补牌数等）；  
  - 公共牌特征（如牌面强度、是否同花面/顺子面）；  
  - 游戏轮次（如 Preflop/Flop/Turn/River 的 one-hot 或标量）；  
  - 下注统计（如当前轮最大下注、总下注等），并用 max_stack 或底池相关量归一化到 [0,1]；  
  - 动作相关（如是否有人加注、是否有人全押等）。  
  上述维数、命名与具体公式可根据游戏（如德州扑克）调整；核心是“基于博弈先验的显式特征 + 与原始信息状态拼接”。

- **优势目标（遗憾值）的尺度归一化与裁剪**：在子进程生成优势样本时，将遗憾值除以归一化尺度（如 max_stack）缩放到 O(1)，并对归一化后的值做裁剪（如限制在 [−C, C]），再写入样本；主进程训练优势网络时使用该归一化后的目标，减少梯度爆炸与训练震荡。

- **金额特征的归一化**：信息状态或先验特征中的金额类维度（如下注统计），在拼接前或在前向计算前，用 max_stack 或底池大小等做线性或对数归一化到有界区间，保证不同游戏配置下尺度一致。

### 4.4 策略网络（及优势网络）训练的多维度分层采样方法

- **分层维度**：（1）**时间维度**：以当前迭代次数 current_iteration 和窗口 new_sample_window 为参数，将样本划分为新近样本（迭代次数落在 [current_iteration − new_sample_window, current_iteration] 内）与历史样本（其余）；当 new_sample_window 为 0 时，新近样本即迭代次数等于 current_iteration 的样本。（2）**重要性维度**：根据样本的重要性度量划分层级，例如遗憾值幅度（绝对值或范数）、优势函数绝对值、对策略/优势网络更新贡献度的估计（如梯度范数、TD 误差类度量等）；可将样本划分为高重要性层与低重要性层，或按重要性分桶形成多档。上述时间与重要性维度可单独使用，也可组合使用（如先按时间分新/老，再在每层内按重要性分高/低），形成多维分层结构。

- **分层采样步骤**：给定本次训练所需的样本数 num_samples 及各层目标占比（如新近样本占比 new_sample_ratio、高重要性样本占比等）：  
  - 在经验缓冲区中，根据样本的迭代次数字段与可选的重要性字段，将索引划分为各层（如新近样本索引集、历史样本索引集；或进一步按重要性划分子层）；  
  - 按设定比例计算各层目标抽取数（如 desired_new = min(floor(num_samples × new_sample_ratio), 新近样本总数)，desired_old = num_samples − desired_new，且不超过历史样本总数；若某层不足则由其他层补足）；  
  - 从各层索引集中无放回随机抽取对应数量，合并后打乱顺序，再按索引从缓冲区取出对应样本形成训练批次；或采用**重要性加权的分层采样**：在保证各层最低占比的前提下，对高重要性样本适当提高被抽中的概率（如按重要性权重进行加权随机抽样），再合并形成训练批次；  
  - 将该批次用于策略网络和/或优势网络的本轮训练。

- **效果**：通过时间维度保证新近样本占比、减轻分布漂移；通过重要性维度提高对关键样本（大遗憾值、高贡献度）的利用效率，降低训练抖动、提升稳定性与收敛性。

以上四部分（并行训练架构、子进程队列供给经验缓冲区、输入特征先验与设计、分层采样）可在同一系统中同时使用，形成完整的 DeepCFR 训练方案。

## 5、本发明的关键点和欲保护点是什么？

1. **并行训练架构**：主进程负责 GPU 上的策略网络与优势网络训练及经验缓冲区管理，多个子进程负责 CPU 上的游戏树遍历与样本生成；二者通过样本队列与参数同步队列连接，无 Parameter Server；经验缓冲区的样本由主进程从队列取出后写入，实现遍历与训练的并行化。

2. **子进程队列替代经验缓冲区样本的方法**：经验缓冲区中的样本由子进程通过队列供给——子进程将样本放入样本队列，主进程从队列取出并写入经验缓冲区；队列满时子进程侧采用**一部分 FIFO、一部分随机替换**的混合替换策略（如一部分情况丢弃最旧、另一部分情况随机选择位置丢弃后再放入），兼顾时序性与多样性、避免阻塞；经验缓冲区若已满也可采用混合淘汰（一部分 FIFO、一部分随机替换）。可选地按玩家分设优势样本队列与策略样本队列，主进程批量收集并标记迭代次数及可选的重要性度量。

3. **输入特征先验知识与设计**：从游戏配置字符串自动解析归一化基准（如 max_stack）；在原始信息状态上拼接基于博弈规则的先验特征（牌力、阶段、成牌/听牌、下注统计、动作特征等）；对金额特征与优势目标（遗憾值）按基准做归一化与可选裁剪，使输入与目标尺度稳定、先验知识显式参与训练。

4. **策略网络（及优势网络）训练的多维度分层采样**：从经验缓冲区采样时，可按多维度对样本分层——时间维度（按迭代次数/时间戳划分新近样本与历史样本）、重要性维度（按遗憾值幅度、对策略更新贡献度等划分高/低重要性层）；按设定比例从各层分别抽取再合并打散，或采用重要性加权的分层采样，形成训练批次，保证新近样本与高重要性样本占比，减轻分布漂移并提升样本利用效率。

代理人撰写权利要求时，可对上述 1～4 分别或组合进行概括与分层，以方法权利要求、装置/系统权利要求等形式保护。

## 6、用推理方式推导出本发明的优点

- **并行训练**：游戏树遍历与网络训练分属不同进程并行执行，CPU 与 GPU 同时利用率提高，训练吞吐量显著提升（实践中可达数倍量级）；通过队列与经验缓冲区解耦，主进程逻辑清晰，无需 Parameter Server。

- **子进程队列供给经验缓冲区**：样本来源明确（队列→主进程→缓冲区），易于实现与调试；队列与缓冲区满时采用一部分 FIFO、一部分随机替换的混合策略，兼顾时序性与多样性、避免阻塞与死锁，队列长度可控；主进程统一标记迭代次数及可选的重要性度量，便于多维度分层采样。

- **输入特征先验与尺度归一化**：先验特征显式引入牌力、阶段、下注等信息，降低学习难度、提高样本效率；从配置解析归一化基准使多配置与多游戏可复用同一套特征与目标尺度；遗憾值归一化与裁剪减少梯度爆炸与训练震荡，提升稳定性。

- **多维度分层采样**：新近样本与高重要性样本占比得到保证，减轻分布漂移带来的训练抖动，同时提高对关键样本的利用效率，策略网络与优势网络训练更稳定，收敛行为更可预期。

## 7、针对 4 中的技术方案，是否还有别的替代方案同样能完成发明目的？

- **并行与通信**：可用多线程或协程在单进程内模拟“遍历与训练”的并发；或使用共享内存传递大块样本、队列仅传参数；分布式环境下可将主进程与子进程部署到多机，队列改为分布式消息队列，逻辑与单机多进程一致。

- **队列与缓冲区**：队列满时采用一部分 FIFO、一部分随机替换的混合策略（如按比例或随机决策在 FIFO 与随机替换间切换）；经验缓冲区除 Reservoir/RandomReplacement 外，可采用混合淘汰（一部分 FIFO、一部分随机替换）、固定大小 FIFO、优先经验回放等结构，只要主进程从队列取样本再写入即可。

- **先验特征与归一化**：先验特征的具体维度与公式可根据游戏调整；归一化基准除 max_stack 外可引入盲注、底池等；目标裁剪可采用其他有界函数（如 tanh）；先验特征也可部分放在主进程侧计算后再写入缓冲区。

- **分层采样**：分层维度除时间（迭代次数/时间戳）外，可采用重要性（遗憾值幅度、贡献度、优先级等）、玩家、游戏阶段等多维；可按设定比例从各层抽取，或采用重要性加权的分层采样；比例与窗口可固定或随训练进度动态调整；仅对策略网络或仅对优势网络使用分层采样、另一网络仍随机采样，也属替代方案。
