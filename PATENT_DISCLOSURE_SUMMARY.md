## 1、本发明要解决的技术问题

1. **并行训练**：DeepCFR 中游戏树遍历（CPU 密集）与网络训练（GPU 密集）若串行执行，CPU 与 GPU 无法同时满载。如何设计并行架构使遍历与训练解耦、并行执行，提高训练吞吐量？
2. **经验缓冲区样本来源**：传统为单进程本地遍历后直接写入；多子进程时若直接写共享缓冲区则锁竞争复杂。如何设计“子进程→队列→主进程→缓冲区”的供给方式，且队列或缓冲区满时采用**一部分 FIFO、一部分随机替换**的混合替换策略，兼顾时序性与多样性、避免阻塞？
3. **输入特征与训练目标尺度**：金额特征与遗憾值随配置与牌局变化大，易导致训练不稳定。如何从游戏配置解析归一化基准，并设计先验特征与尺度归一化/裁剪以提升稳定性？
4. **采样分布**：纯随机采样时新近或高重要性样本占比可能过低，导致分布漂移与训练抖动。如何设计**多维度分层采样**（如时间/迭代次数 + 重要性如遗憾值幅度、对策略更新贡献度），按比例从各层抽取或采用重要性加权的分层采样，保证新近样本与高重要性样本占比、减轻分布漂移并提升样本利用效率？

## 2、技术背景与现有方案

### 2.1 公知技术（摘要）

- **DeepCFR**：每轮迭代对博弈树遍历（Monte Carlo），利用优势网络采样并计算反事实值与遗憾值，生成优势/策略样本，再更新策略网络与优势网络；训练同时包含“游戏树遍历（CPU）”与“网络训练（GPU）”。
- **经验缓冲区与采样**：遍历得到的样本存入经验缓冲区，训练时从缓冲区采样；传统单进程实现为“本地产生、本地写入”。
- **多进程与队列**：单机多核下可用多进程 + 队列实现生产者-消费者式并行。
- **应用场景**：本发明特别适用于**德州扑克**及同类不完全信息博弈。下文 2.2、2.3 涵盖所有检索到的专利与文献，供代理人对比。

## 3、现有技术缺点与发明目的

- **推理一（并行训练）**：单进程 DeepCFR 中遍历与训练串行，CPU/GPU 无法同时满载；通用分布式（Actor-Learner+PS）面向环境交互非博弈树且 PS 增加复杂度；Ray 版 DeepCFR 未明确单机多进程+队列的解耦方式。**目的**：提供面向 DeepCFR 的并行训练架构，主进程 GPU 训练与缓冲区管理、多子进程 CPU 遍历，经队列与共享缓冲区连接，提高训练效率。
- **推理二（经验缓冲区样本来源）**：传统为本地遍历直接写入；多进程下子进程直接写共享缓冲区则锁竞争复杂；无“子进程→队列→主进程→缓冲区”则无法稳定注入样本；队列满时仅单一 FIFO 或单一随机替换各有不足。**目的**：子进程将样本放入队列、主进程取出写入经验缓冲区；队列满时子进程侧采用**一部分 FIFO、一部分随机替换**的混合策略，兼顾时序性与多样性、避免阻塞；经验缓冲区若已满也可采用混合淘汰。
- **推理三（输入特征与目标尺度）**：金额与遗憾值范围随配置与牌局变化大，固定归一化难兼顾；仅用原始信息状态不引入先验会加大学习难度。**目的**：从游戏配置解析归一化基准（如 max_stack）；设计并拼接先验特征（牌力、阶段、下注统计等）；对遗憾值等按基准归一化并可选裁剪，提升稳定性与收敛性。
- **推理四（采样分布）**：纯随机采样时新近或高重要性样本占比可能过低，分布漂移、训练抖动。**目的**：采用**多维度分层采样**——除按时间/迭代次数划分新近与历史样本外，还可按重要性（遗憾值幅度、对策略更新贡献度等）划分高/低重要性层；按比例从各层抽取或采用重要性加权的分层采样，保证新近样本与高重要性样本占比、减轻分布漂移并提升样本利用效率。

## 4、本发明技术方案（摘要）

建议配套附图：系统架构图、主进程/子进程流程图、多维度分层采样流程图（含时间与重要性维度）、先验特征与归一化框图（中文、黑白）。

- **4.1 并行训练架构**：主进程（GPU）负责从经验缓冲区采样、训练策略网络与优势网络、将参数经参数同步队列下发给子进程、管理经验缓冲区（从样本队列取样本写入、从缓冲区采样）；多个子进程（CPU）从参数同步队列取参数、本地游戏树遍历（Monte Carlo）、生成优势/策略样本并放入样本队列。样本队列与参数同步队列连接主/子进程，经验缓冲区由主进程管理；数据流为“子进程遍历→入队→主进程取样本写缓冲区→分层采样→GPU 训练→参数下发→子进程更新”，实现遍历与训练流水线并行。
- **4.2 子进程队列替代经验缓冲区样本**：经验缓冲区样本由“本地产生”改为“子进程经队列供给”；主进程从样本队列批量取样本写入经验缓冲区并标记迭代次数及可选的重要性度量；队列满时子进程侧采用**一部分 FIFO、一部分随机替换**的混合策略（如一部分情况弃最旧、另一部分情况随机选择位置丢弃再放入），兼顾时序性与多样性、避免阻塞；经验缓冲区若已满也可采用混合淘汰（一部分 FIFO、一部分随机替换）。
- **4.3 输入特征先验与归一化**：从游戏配置字符串解析 max_stack 等归一化基准；在原始信息状态上拼接先验特征（起手/当前牌力、成牌类型、听牌、公共牌特征、游戏轮次、下注统计归一化、动作相关等）；对遗憾值按 max_stack 归一化并裁剪（如 [−C,C]）；金额特征用 max_stack 或底池归一化到有界区间。
- **4.4 多维度分层采样**：分层维度包括（1）**时间维度**：按当前迭代与窗口将样本分为新近样本与历史样本；（2）**重要性维度**：按遗憾值幅度、对策略/优势网络更新贡献度等划分高/低重要性层。按设定比例从各层分别抽取再合并打乱，或采用**重要性加权的分层采样**（在保证各层最低占比的前提下对高重要性样本适当提高采样概率），形成训练批次，用于策略网络与优势网络，保证新近样本与高重要性样本占比、减轻分布漂移并提升样本利用效率。

## 5、关键点与保护点

1. **并行训练架构**：主进程 GPU 训练与缓冲区管理、多子进程 CPU 遍历与样本生成，经样本队列与参数同步队列连接，无 Parameter Server；经验缓冲区样本由主进程从队列取出后写入；实现遍历与训练并行化。
2. **子进程队列替代经验缓冲区样本**：子进程将样本放入样本队列、主进程取出写入经验缓冲区；队列满时子进程侧采用**一部分 FIFO、一部分随机替换**的混合策略；经验缓冲区若已满也可采用混合淘汰；可选按玩家分优势/策略队列；主进程批量收集并标记迭代次数及可选的重要性度量。
3. **输入特征先验与设计**：从游戏配置解析归一化基准；在原始信息状态上拼接先验特征（牌力、阶段、成牌/听牌、下注统计、动作特征等）；对金额特征与遗憾值按基准归一化与可选裁剪。
4. **多维度分层采样**：从经验缓冲区按多维度分层（时间：迭代次数/时间戳；重要性：遗憾值幅度、对策略更新贡献度等），按比例从各层抽取或采用重要性加权的分层采样，形成训练批次，保证新近样本与高重要性样本占比、减轻分布漂移并提升样本利用效率。

代理人可对上述 1～4 分别或组合撰写方法权利要求与装置/系统权利要求。

## 6、优点

- **并行训练**：遍历与训练分进程并行，CPU 与 GPU 利用率同时提高，训练吞吐量显著提升；队列与缓冲区解耦，无需 Parameter Server。
- **队列供给缓冲区**：样本来源清晰（队列→主进程→缓冲区），易实现与调试；一部分 FIFO、一部分随机替换的混合策略兼顾时序性与多样性、避免阻塞与死锁；主进程统一标记迭代及可选的重要性便于多维度分层采样。
- **先验特征与归一化**：先验显式引入牌力、阶段、下注等信息，降低学习难度；配置解析归一化基准使多配置可复用；遗憾值归一化与裁剪减少梯度爆炸与震荡。
- **多维度分层采样**：新近样本与高重要性样本占比有保证，减轻分布漂移与训练抖动，提高对关键样本的利用效率，收敛更稳定。

## 7、替代方案

- 并行与通信：单进程内多线程/协程模拟并发；共享内存传大块样本、队列仅传参数；多机部署时队列改为分布式消息队列。
- 队列与缓冲区：队列满时采用一部分 FIFO、一部分随机替换的混合策略；经验缓冲区可采用混合淘汰、固定 FIFO、优先经验回放等。
- 先验与归一化：先验维度与公式可按游戏调整；归一化基准可加入盲注、底池；裁剪可用其他有界函数；部分先验可在主进程侧计算后写入缓冲区。
- 分层采样：分层维度除时间（迭代/时间戳）外可采用重要性（遗憾值幅度、贡献度、优先级）、玩家、游戏阶段等；可按比例从各层抽取或重要性加权；比例与窗口可动态调整；可仅对策略或优势网络使用分层采样。
